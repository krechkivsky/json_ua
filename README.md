Валідація містобудівної документації (Наказ №56) — JSON Schema & GIS Tools

Огляд проекту
Цей репозиторій містить повну технічну реалізацію структури геоданих містобудівної документації на місцевому рівні згідно з Наказом Мінрегіону №56 від 22.02.2022.

Проект вирішує критичну проблему галузі: відсутність офіційного механізму суворої валідації при переході від формату XML (XSD) до JSON. Мною розроблено архітектуру JSON-схем, яка забезпечує «XSD-подібний» контроль за цілісністю даних, що є критично важливим для інтеграції в муніципальні геоінформаційні системи (МБКД).

Основні компоненти
schema.json: Головна схема, що описує понад 100 класів об'єктів та їхні атрибутивні зв'язки.

common.json: Модульна бібліотека базових типів, геометрій (GeoJSON) та глобальних властивостей.

Інтегровані довідники: Повна імплементація системних переліків (Enums) для функціонального призначення територій, категорій земель та інших класифікаторів.

Технічні переваги
Сувора валідація типів: Автоматичне відстеження відповідності типів полів (LongInteger, Double, GUID, String) на рівні вхідного файлу.

Топологічна чистота: Схема підготовлена для роботи з алгоритмами валідації QGIS/GEOS, що дозволяє виявляти не лише накладання (Overlaps), а й складні топологічні дефекти, такі як вузькі щілини (Slivers) та розриви суміжності.

Автоматизація опису метаданих: Використання oneOf та константних класифікаторів для автоматичного визначення класів об'єктів без необхідності ручного задання типів.
